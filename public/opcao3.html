Avançados!

Desafio: Criar um sistema de simulação de um jogo de cartas. Crie classes para representar um baralho (Baralho), cartas (Carta), jogadores (Jogador) e o jogo (Jogo). Implemente métodos para embaralhar o baralho, distribuir cartas aos jogadores e realizar rodadas de jogo. Utilize herança e polimorfismo para representar diferentes tipos de cartas e jogadores com estratégias diferentes. Implemente regras complexas do jogo, como a contagem de pontos e condições de vitória, usando padrões de projeto como o Strategy e o Observer para gerenciar eventos durante o jogo.

João Silva

Professor de Programação Orientada a Objetos

Desafio: Criar uma aplicação web interativa utilizando tecnologias modernas. Desenvolva um sistema de gerenciamento de tarefas (todo list) com funcionalidades como adicionar, editar, excluir e marcar como concluída. Utilize uma arquitetura de backend RESTful com Node.js e Express para criar APIs robustas que suportem as operações CRUD. Implemente autenticação e autorização usando JWT (JSON Web Tokens) para proteger rotas e garantir que apenas usuários autenticados possam acessar determinadas funcionalidades. Utilize um banco de dados NoSQL como MongoDB para armazenar as tarefas, aproveitando as vantagens de flexibilidade e escalabilidade oferecidas por este tipo de banco de dados.

Marcelo Mathias

Professor de Desenvolvimento Web

Desafio: Implementar uma árvore binária de busca. Comece criando uma classe No para representar cada nó da árvore, contendo campos chave, esquerda, direita e pai. Implemente métodos para inserir (inserir), buscar (buscar), remover (remover) e percorrer (pre_ordem, em_ordem, pos_ordem) na árvore. Otimize os métodos de busca e remoção para lidar com casos complexos, como remoção de nós com dois filhos, e implemente balanceamento automático (por exemplo, AVL ou Red-Black) para manter a árvore balanceada após operações de inserção e remoção

Valter Souza

Professor Estrutura de Dados

Desafio: Criar um sistema completo para uma biblioteca virtual. Comece criando um banco de dados chamado Biblioteca com tabelas Livros (com id, titulo, autor, ano_publicacao) e Usuarios (com id, nome, email). Em seguida, desenvolva uma tabela Emprestimos com as colunas id, usuario_id, livro_id, data_emprestimo e data_devolucao. Implemente procedimentos armazenados (stored procedures) para realizar operações complexas como empréstimos automáticos com verificação de disponibilidade de livros. Utilize transações para garantir a integridade dos dados durante operações de empréstimo e devolução simultâneas. Por fim, crie índices apropriados para otimizar consultas frequentes, como buscar todos os livros emprestados por um usuário específico.
